# The dependencies below use floating version ranges and carry NO hashes, so a
# compromised or yanked release on PyPI could be pulled into a build. For
# reproducible, tamper-evident deploys, generate a fully-pinned + hashed
# lockfile and install from it instead of from these ranges:
#
#   pip install pip-tools
#   pip-compile --generate-hashes --output-file requirements.lock requirements.txt
#   pip install --require-hashes -r requirements.lock
#
# Regenerate the lockfile whenever a range here changes, and commit it.
# (These ranges are intentionally left in place: blindly hard-pinning untested
#  versions can break deploys — pin via the generated lockfile, build-tested.)
#
# Base-image hardening (Dockerfile / docker/full-stack/Dockerfile): the images
# use moving tags (ubuntu:22.04, python:3.11-slim) which can change underfoot.
# Digest-pin them so a build is byte-reproducible, e.g.:
#   FROM ubuntu:22.04@sha256:<digest>
# Resolve the current digest with (requires registry/network access):
#   docker pull ubuntu:22.04 && \
#     docker inspect --format='{{index .RepoDigests 0}}' ubuntu:22.04
